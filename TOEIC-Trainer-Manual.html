🎭 ロールプレイ

⚠️ Azure Speech APIキーが必要です（HOME画面の「🎭 Azure設定」から入力）

フロー

Part 3 回答 → 🎭 ボタン → 役割選択 → 会話開始

🎤

スコア表示

指標説明

🔍 品質管理システム

選択肢品質チェック（パート別基準）

パート良いdistractorC判定の条件
Part 1文法的に正しい文だが写真と合わない文として壊れている
Part 2正しい応答だが質問と噛み合わない
断片応答("Down the hall"等)はOK		gibberish、完全に無関係
Part 3/4文脈上もっともらしいが事実と違う話題と無関係、意味不明
Part 5 品詞意図的に品詞が違う（正常設計）—
Part 5 語彙同品詞で意味/コロケーションが違う品詞が混在（消去法で解ける）
Part 6Part 5同様 + 文挿入問題パッセージと無関係
Part 7本文の情報だが質問の答えではない本文と完全に無関係

アイテム判定（多数決方式）

Part 3の3問セットで Q1:A Q2:A Q3:C → アイテム全体は A（Cは少数）。削除は全設問Cのアイテムのみ対象。

出題優先順位

qualityPriorityShuffle() により A/B → C の順で出題。品質の低い問題は後回し。

📈 IRT推定の仕組み

各パート直近50問を取得:
  Part 1: 直近50問 → effectiveDifficulty(h) → diffToLevel → IRT推定
  ...
  Part 7: 直近50問 → ...

effectiveDifficulty(h) = max(h.difficulty, LEVEL_FLOOR[h.level])
  LEVEL_FLOOR: beginner=0, intermediate=450, advanced=650

→ L(P1-4)合算 → /2 → Lスコア (5-495)
→ R(P5-7)合算 → /2 → Rスコア (5-495)
→ 合計 = 推定TOEICスコア

🔒 セキュリティ

JSONインポートのXSS防止

外部から取得したJSONファイルをインポートする場合のXSS攻撃を防止しています。

対策説明
safeImgUrl()imgUrlがdata:image/またはblob:で始まることを検証
esc()全テキスト出力をHTMLエスケープ（ctxBlock label含む）
インポート時検証ストリーミング/クラシック/模試の全インポート経路でサニタイズ

APIキー管理

Gemini/AzureキーはlocalStorageに保存。個人利用では十分安全。配布時はXSS防止済みのため、悪意あるJSONからの窃取は不可能。
